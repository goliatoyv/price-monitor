# BRD — Price Scout: Агент мониторинга цен для баера

**Версия:** 1.0  
**Дата:** 12.05.2026  
**Статус:** Согласовано ✅

---

## 1. Описание продукта

Веб-сервис для баера, который отслеживает цены на товары на любых сайтах, уведомляет в Telegram-канал/группу когда цена упала или достигла целевого значения, и позволяет управлять статусами закупок.

---

## 2. Контекст и цели

**Проблема:** Баер вручную мониторит десятки сайтов в поисках выгодных цен — это медленно и неэффективно при большом списке позиций.

**Цель MVP:** Автоматизировать мониторинг цен, получать мгновенные уведомления при снижении, управлять статусами позиций через единый интерфейс.

## 4. Функциональные требования (MVP v1)

### M1 — Управление позициями (Watch List)

| ID | Требование |
|---|---|
| M1.1 | Добавить позицию по URL конкретного товара — система автоматически определяет сайт, запускает парсер, заполняет название и текущую цену |
| M1.2 | Для каждой позиции задавать: target price, размер/вариант, заметки, приоритет (🔥 / обычный) |
| M1.3 | Поддержка любых сайтов — универсальный LLM-парсер адаптируется к структуре каждого сайта автоматически |
| M1.4 | Приоритетные сайты с точными парсерами: joesnewbalanceoutlet.com, en.zalando.de |

### M2 — Мониторинг и алерты

| ID | Требование |
|---|---|
| M2.1 | Автоматическая проверка цен по расписанию — каждые 4 часа (настраивается) |
| M2.2 | Алерт при падении цены ниже target price |
| M2.3 | Алерт при снижении цены от последней зафиксированной (любое падение) |
| M2.4 | Алерт при появлении нужного размера (out-of-stock → in-stock) |
| M2.5 | Уведомления в Telegram: поддержка канала (@channel), группы (-100xxx) и личного чата одновременно |
| M2.6 | Inline-кнопка "🛍 Купить сейчас" прямо в Telegram-сообщении |
| M2.7 | Ежедневный дайджест — сводка всех изменений цен за 24 часа |

### M3 — Dashboard (веб-интерфейс)

| ID | Требование |
|---|---|
| M3.1 | Таблица всех позиций: фото, название, сайт, текущая цена, target price, % до цели, тренд (↑↓), статус, дата проверки |
| M3.2 | Строка поиска по всем полям сразу |
| M3.3 | Фильтрация по: статусу, сайту, % до target, приоритету |
| M3.4 | Сортировка по любой колонке |
| M3.5 | Плашка "Горячие сделки" — позиции ниже target или с падением >20% за неделю |
| M3.6 | Мини-график истории цены для каждой позиции |
| M3.7 | Статистика вверху: кол-во позиций ниже target, упавших за 24ч, всего позиций, кол-во сайтов |

### M4 — Статусы позиций

Жизненный цикл:

```
👀 Watching → 🔔 Alert! → ✅ To Buy → 📦 Ordered → 🏠 In Stock
                                ↘
                              ❌ Skipped (пропустить, продолжать следить)
```

## 7. Архитектура универсального парсера

Система НЕ требует изменения кода при добавлении нового сайта. Алгоритм:

1. Пользователь добавляет URL нового сайта
2. Система проверяет таблицу `site_parsers` — есть ли правила для этого домена?
3. **Если нет:** загружает HTML товара → отправляет `<main>` в Claude Haiku → LLM возвращает CSS-селекторы цены, названия, наличия → сохраняет в реестр
4. **Если да:** использует сохранённые селекторы напрямую (быстро, без LLM)
5. **Самовосстановление:** если селектор вернул `null` 3 раза подряд — автоматически перезапускает LLM-анализ и обновляет реестр

**Исключение:** Zalando и другие SPA (React/Vue) рендерят контент через JavaScript. Для них обязателен ScraperAPI.

## 9. MVP scope

### ✅ Входит в v1
- Добавление товара по URL (ручной ввод на сайте)
- Универсальный LLM-парсер + реестр сайтов (самообучение)
- Точные парсеры: JNBO, Zalando (через ScraperAPI)
- Watch list — таблица с ценами, фильтры, поиск
- Target price + алерт при снижении
- Telegram: канал + группа + ежедневный дайджест
- История цен (мини-график на карточке)
- Статусы позиций (watching → to_buy)
- GitHub Actions cron (каждые 4ч)
- Деплой: Supabase + Vercel

### ⏸ v2 — потом
- ROI-калькулятор (цена покупки → маржа → рекомендованная цена продажи)
- Авторизация (multi-user)
- Браузерное расширение (добавить с любой страницы)
- Сканирование категорий (добавить всю страницу листинга)
- AI-рекомендации ("обычно стоит $X на StockX")
- Кросс-сайтовое сравнение одного артикула
- Импорт CSV
- Email-уведомления
